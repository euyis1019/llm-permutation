# 开发需求清单

## 外部设备复现与 benchmark 可移植性

状态：进行中。核心 GPU 实验已经可以外部复现，完整 benchmark 流程仍需兼容性整理。

现有 `experiments/ffn_permutation/` Python 探针支持外部用户传入本地模型路径和独立结果目录，在自己的受支持 NVIDIA GPU 上运行核心实验，并生成属于该设备的 JSON、JSONL 和环境 manifest。脚本同时支持 smoke test 与断点续跑；新结果应写入独立目录，不覆盖仓库中已经版本化的实验记录。

待完善范围：

- 补充独立于 Notebook 轻量依赖的 GPU 环境与模型准备说明；
- 提供一个统一的便捷入口，串联现有核心探针、汇总与完整性检查；
- 清理 benchmark 调度器中的机器绝对路径，并通过新配置覆盖机制保持冻结配置不变；
- 让 benchmark 流程支持外部模型路径、GPU 编号和独立输出目录；
- 在结果 manifest 中记录 GPU、CUDA、关键 Python 包版本、模型标识和运行状态；
- 允许核心 Notebook 选择外部结果目录，用于重建和比较不同设备的图表；
- 增加最小 smoke test，失败时不得把不完整结果标记为成功。

当前限制主要集中在完整 benchmark：其冻结配置和部分调度脚本仍包含原始机器路径，尚未形成开箱即用的跨机器执行入口。这不影响外部用户使用现有探针复现核心 GPU 实验。
