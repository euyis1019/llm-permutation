## 0. 这一轮要干什么

已经确认的事实:permutation 在数学上不改变模型函数,但会改变 GPU 浮点求和的顺序,产生微小数值漂移,漂移偶尔会翻转答案。这一轮要回答的最后一个问题是:这种漂移对模型能力到底意味着什么?

你的预期结论(我认同,已有数据也倾向支持)是:模型计算对这种扰动有冗余。漂移会改变某道题的作答路径,但不改变模型的整体能力水平;看起来像"变好"或"变坏"的 benchmark 分差,只是同一能力水平附近的正常波动。

这一轮实验的目标,是把上面这句话从判断变成检验过的结论,每一步都有数字,并且说清它对外的用处。

## 1. 意义在哪里:三个具体的用处

### 用处一:给"随机扰动找好模型"这类方法定出噪声刻度(对接 RandOpt / ZO)

先复述 RandOpt(Neural Thickets)在干什么:在预训练权重 θ₀ 附近采样几千个随机噪声候选 θ₀+σ·ε(ε 是标准高斯噪声),用一个打分集挑出得分最高的 top-k,主张"预训练模型附近密集存在更强的专家模型"。从它的代码里读出两个关键设置:

- σ 的网格是 {1e-4, 5e-4, 1e-3, 2e-3, 5e-3, 1e-2},噪声按参数原 dtype(bfloat16)直接加到权重上;
- 同一条文献线上,ZO 微调方法的试探步长也在这个量级:MeZO 常用 ε=1e-3,FZOO 常用 ε=1e-4。

我们已经用真实 Qwen3-4B 权重(第 17 层 down_proj)做了一个零成本的预演测量:

| σ | 噪声被 bf16 量化格距直接吞掉的比例 | 实际造成的权重相对变化 |
|---:|---:|---:|
| 1e-4 | 35%(这部分加了等于没加) | 0.46% |
| 5e-4 | 8% | 2.2% |
| 1e-3 | 4% | 4.4% |

(权重的中位数只有 0.0145,bf16 在这个量级的相邻刻度间距约 3e-5,所以 σ=1e-4 的噪声有三分之一比刻度间距还小,加上去被舍入回原值。)

对照我们这边的事实:permutation 对权重的改变严格为 0,却让全模型 logits 产生约 2% 的相对变化,让生成类 benchmark 产生 ±1pp 的波动。于是你提出的那个问题可以被精确测量:

> σ 小到什么程度时,高斯扰动给 logits 带来的改变量,还不如"权重一点没动、只换浮点求和顺序"带来的改变量大?

把这个临界点叫 σ\*(噪声等效半径)。测出 σ\* 之后,"邻域找模型"的图景就有了刻度:

```text
半径严格为 0       : 块内 permutation,逐比特不变(Part 1 收尾)
函数距离 0、路径不同 : 跨块 permutation,logits 漂 ~2%,benchmark ±1pp(已测)
σ 低于 bf16 格距    : 高斯噪声部分被量化吞掉,名义扰动大于实际扰动(上表,已测)
σ < σ*            : 候选与"函数完全相同的模型"在 logits 上不可区分,
                     候选之间的分数差主要来自数值噪声(Part 6 测 σ*)
σ > σ*            : 真正的功能差异从这里开始,这才是 RandOpt 有效的工作区
```

最终产出一张图:横轴是扰动大小(从 0 到 1e-2),纵轴是 logits 漂移和答案翻转率,图上标出 permutation 水平线、bf16 格距区、RandOpt 的 σ 网格、MeZO/FZOO 的 ε 位置。这张图直接回答:RandOpt 的 σ 网格里,哪几档产生的候选差异主要是数值噪声。

在此之上,20 个 permutation 模型作为函数完全相同的候选池,还提供一个零点:在这个池子里"挑最好的"挑出的任何增益都是假象,这一点由数学保证,不是假设。把假象增益曲线做出来(Part 3),任何 RandOpt 式方法声称的小 σ 增益都应先超过这条线,否则等于什么都没找到。

对 ZO 方法另有一个直接可用的结论(Part 7)。ZO 靠比较 L(θ+εu) 和 L(θ−εu) 两次 loss 的微小差值来猜更新方向;我们可以直接测出纯数值路径噪声能让 loss 变多少。这是 ZO 探针信号的噪声下界:探针差值低于它,估出来的方向就是在拟合浮点噪声。bf16 下 ε=1e-4 的探针是否经常落在这个下界以下,一测便知。

### 这项工作的意义,用两句话说

有人可能会问:"人家做参数扰动又不做 permutation,你测的噪声跟人家有什么关系?"回答:

> Permutation 不改变模型函数,所以它引起的全部变化就是 BF16 前向计算自带的数值噪声,我们只是借它把这层噪声的大小测了出来(logits 约 2%,生成类 benchmark ±1pp)。任何扰动方法评估候选效果用的也是同一种前向计算,所以这个数就是他们测量结果的精度下限:声称的效果比它小,测到的就是噪声,和做不做 permutation 无关。

Part 6 的小 σ 实验直接检验这句话:预期高斯扰动的 σ 再小,logits 响应也降不到这条线以下(如果降下去了,就如实缩小下限的适用范围)。另一个用已有数据就能说明的事实:RandOpt 用 vLLM 连续批处理给候选打分,而默认 vLLM 配置同权重重跑 GSM8K 有 15/100 答案翻转,候选分数本身就带着这个量级的测量噪声。

### 用处二:把"鲁棒性"从一句话变成一条数字链

现在说"模型是鲁棒的"是空话。做完这一轮,可以说成一条每级都有实测数字的衰减链:

```text
单层里 35–47% 的输出被挪动 1 个 BF16 刻度(挪动比例不小)
  → 全模型 logits 相对变化只有 ~2%
  → 下一个词的选择只有 ~2% 被翻转
  → 翻转集中在"前两名几乎并列"的犹豫决策点(待本轮量化)
  → benchmark 只在长生成任务上留下 ≤1pp、方向不可利用的波动(待本轮判定)
```

"冗余"的定量含义是:绝大多数决策点上,第一名领先第二名的优势(margin)远大于数值漂移能撬动的量级,所以漂移被一路吸收。这条链就是"activation 对 weight 扰动有冗余"的直接展示。

### 用处三:工程结论,严格零扰动的置换类(已基本完成,本轮收尾)

前两轮已确认"只在对齐 8-块内部重排"在 vLLM 上单层逐比特不变。本轮把它推到最终交付物:整个引擎 36 层跑完 logits 逐比特不变,benchmark 逐题一字不差。这给鲁棒性谱定下严格的零端点,也是可以单独使用的工程结论:想改 FFN 通道顺序又不想改变模型任何行为时,存在经过验证的安全操作域。

## 2. 实验列表

共 8 个部分。需要 GPU 的是 Part 0/1/2/6,合计约 15.5 小时(GPU 0 共享,显存占用 0.28);Part 3/4/5/7 是对已有数据或顺带记录的数据做纯分析,零额外 GPU 成本。

### Part 0:重复运行基线(你上次提出的那个)

- 问题:同一个模型、同一个问题,跑两次,logits 是不是逐比特一样?后面所有逐比特判定都依赖这一条。
- 操作:不做任何置换的原模型,用冻结的确定性配置(batch-invariant 等),开两个独立进程各跑一遍同样的 32 个 prompt,各自保存最后一个 token 的 logits,然后逐比特比对。
- 产出:32 个 prompt 中逐比特一致的数量(期望 32/32);若不一致,给出差异大小。
- 结果怎么读:32/32 说明比较基准成立,Part 1 的判定有效。不到 32/32 说明推理栈本身有 run 间噪声,Part 1 的所有逐比特判据要降级为"差异不超过 run 间噪声",先查清这个再继续。
- 已知背景:上一实验在生成结果层面查过重复运行(base 模型 6 个 benchmark 对错全部一致,但 MBPP+ 有 5 条文本差异;instruct 的 GSM8K 有残留噪声)。也就是说,长文本生成已知不是 100% 逐 run 一致,但单步 logits 从未直接验证过,这正是本 part 补的洞。
- 成本:约 0.5 小时。

### Part 1:块内置换从单层推到整个 benchmark

- 问题:块内置换的逐比特不变,目前只在单层矩阵乘上验证过。整个模型 36 层跑完、乃至整个 benchmark 跑完,还是零吗?
- 操作,分两步:
  1. 引擎 logits 级:4 个模型变体(不置换 / 全 36 层块内重排 / 全 36 层奇对齐换位,预测非零 / 全 36 层全局随机,预测非零)各跑同样 32 个 prompt,和不置换版比 logits。
  2. benchmark 级:3 个真实 checkpoint(块内重排 / 奇对齐 / 一个全局随机对照)在 Base 模型上各跑全部 6 个 benchmark,和 baseline 逐题比对。比的是每道题的作答,不是平均分。
- 产出:块内重排臂的逐比特一致 prompt 数和逐题一致样本数(期望全部);奇对齐臂和随机臂的漂移大小与逐题不一致率对比。
- 结果怎么读:块内臂全零,则"零代价置换类"从矩阵乘到 benchmark 全链条成立。块内臂不是全零,说明引擎里还有别的对通道顺序敏感的算子,要定位它(这本身也是发现)。奇对齐臂与随机臂相当,则再次确认"跨块是开关不是斜坡"在 benchmark 层也成立。
- 成本:约 6 小时(主要是 3 checkpoint × 6 benchmark)。
- 说明:这部分沿用 v1.1 已预注册的 2b/3b 设计,不新增内容。上轮因 torch T=1 的例外触发停机没跑成,但那个例外只涉及 PyTorch 单 token 路径,与这里用的 vLLM 栈无关,预测不变。

### Part 2:logits 漂移和答案翻转率挂钩

- 问题:用处二那条数字链里,"logits 变化多少"和"benchmark 行为变化多少"之间还没有定量关系。漂移大的配置,答案翻转是不是就多?
- 操作:上一实验已经跑过 8 个不同强度的置换配置(只动第 0 层 / 只动第 17 层 / 只动最后一层 / 动前 6 层 / 动前 18 层 / 动全部 36 层 / 相邻交换 / 反转),benchmark 逐题结果都在盘上。现在只补测每个配置在同样 32 个 prompt 上的 logits 漂移大小(每个配置加载一次引擎,共 8 次),然后画图:横轴是 logits 漂移,纵轴是已有的逐题不一致率。
- 产出:8 个点的散点图和相关系数;另外用上游已有 logits 数据统计被翻转 token 的 margin 分布,看翻转是否集中在前两名几乎并列的位置。
- 结果怎么读:单调关系成立(预注册预测:等级相关 ρ≥0.9),则可以从 logits 漂移预测行为扰动的量级,数字链补全。不成立,说明行为扰动还受漂移大小之外的因素支配(比如注入位置),这本身修正认知。
- 成本:约 1.5 小时。

### Part 3:在 20 个功能等价的模型里挑最好的,能挑出多大假提升

- 问题:在 20 个功能完全等价的模型里挑表现最好的,能"挑"出多大的提升?换一批题之后,这个提升还剩多少?
- 操作(纯分析,零 GPU):
  1. 把 GSM8K 的 500 道题按题号奇偶分成 A、B 两半,每半 250 题;
  2. 在 A 半上给 20 组随机置换排名,挑出表现最好的一组,模拟"用打分集挑模型";
  3. 看这组随机置换在 B 半上的表现:比 20 组的平均水平好多少?比 baseline 好多少?
  4. 同样流程对每个生成类 benchmark 做一遍,并推广到"挑 top-k 取平均"(k=1,2,3,5),得到完整的对照曲线:A 半上看到的表观提升,对 B 半上剩下的真实提升(预期约等于 0)。
- 产出:一张表或图,列出每个 benchmark、每个 k 的挑选前表观增益和换题后真实增益。
- 结果怎么读:B 半增益回落到约 0(预注册预测),证明这个量级的挑模型增益是纯假象,这条对照曲线可以直接拿去比对 RandOpt 类方法的报告值。B 半增益不回落,说明数值路径的差异产生了可迁移的能力差异,冗余性结论要改写,这个方向的意外同样如实报告。

### Part 4:全负和全正的方向从哪来,回归均值检验

- 问题:20 组随机置换在 Instruct 上全是负,在 Base 上全是正。这看起来像 permutation 有方向性,和你的直觉矛盾。怎么解释?
- 候选解释(可检验):permutation 本身没有方向。20 个置换模型的平均分才代表这个数值邻域的真实水平,方向只取决于 baseline 自己在邻域里的位置。Instruct 的 baseline 恰好在这套题上运气好(很多犹豫题恰好蒙对),任何扰动都倾向打破运气,于是看起来全负;Base 恰好运气差,任何扰动平均把它拉回正常水平,于是看起来全正。这不是 permutation 在推模型往哪走,是回归均值。
- 操作(纯分析,零 GPU):把每道题按"baseline 对不对"和"20 组随机置换里有多少组答对"分类,把总分差拆到各类题上。若 Instruct 的 −0.79pp 主要来自"baseline 对、但多数置换组错"的题(幸运题被扰动打破),Base 的 +0.49pp 主要来自"baseline 错、但多数置换组对"的题(不幸题被扰动修复),回归均值就成立。另外检验每组随机置换具体翻转哪些题、不同置换组之间是否重叠,预测是翻来翻去都是同一批不稳定题,但每组翻转的方向组合不同。
- 产出:分差的逐类分解表;跨 seed 翻转题集重叠率。
- 结果怎么读:成立,则你的直觉被证实,并且有了机制解释:所谓方向是 baseline 在自己邻域里的位置,不是扰动的指向,这直接支撑冗余性结论。不成立(比如分差来自大量稳定翻转的题),说明存在系统性的方向效应,结论要相应修改。

### Part 5:免费的多样性有没有用,多数票检验(顺带的加分项)

- 问题:20 个模型在犹豫题上给出了不同答案,这算一种免费的多样性(不用采样,不用加噪声,函数还完全等价)。把它们的答案做多数票,能不能比单个模型好?
- 操作(纯分析,零 GPU):对能无歧义提取答案的 benchmark(GSM8K 的数字答案、三个选择题 benchmark 的选项),每道题取 20 组随机置换答案的多数票,算多数票的分数,再与单个模型及 20 组随机置换的平均分比较。
- 产出:多数票对单模型的分差,及其与已知运行噪声地板的对比。
- 结果怎么读:多数票没有稳定优势(预注册的保守预测),说明这种多样性是纯噪声级的,进一步支持波动不可利用。多数票有稳定优势,则是意外的正面发现,相当于不用温度采样的自一致性投票。两个方向都有话可说。

### Part 6:测出噪声等效半径 σ*(用处一的核心新实验)

- 问题,两问合一:(a) 高斯扰动的 σ 小到什么程度,其 logits 改变量就降到和 permutation 纯数值噪声一个水平?RandOpt 的 σ 网格有几档落在这个区间?(b) 更根本的:σ 趋向 0 时,logits 响应是连续降到零,还是停在一个不再下降的平台上?(b) 直接检验"精度下限"那句话。
- 操作,三个层级:
  1. 权重级(零 GPU,已做预演):对每档 σ,统计噪声被 bf16 量化格距吞掉的比例、实际实现的权重相对变化。扩展到全模型所有权重矩阵,CPU 上分层统计即可。
  2. logits 级(GPU,核心):σ 网格取 {1e-6, 3e-6, 1e-5, 3e-5} 加上 {1e-4, 5e-4, 1e-3, 2e-3, 5e-3, 1e-2}。前四档低于 RandOpt 网格,专测小 σ 极限;后六档就是 RandOpt 的网格。每档 3 个噪声 seed,按 RandOpt 的方式给全模型权重加噪声(内存中加,不落盘),用与 Part 0/1 相同的 32 个 prompt 测 logits 相对漂移。10×3=30 个点画成"σ 对 logits 漂移"曲线,叠加 permutation 全 36 层的漂移水平线(约 2%)。曲线与水平线的交点就是 σ\*;曲线在小 σ 端的形状(连续下降,还是平台加突降)回答问题 (b)。
  3. 行为级(GPU,可选加做):取两档 σ(最低有效档和最接近 σ\* 的一档)各 5 个随机种子,只跑 GSM8K,并与 20 组随机置换比较三件事:分数波动幅度是否同量级,翻转的题是不是同一批不稳定题,A/B 半分挑选候选的增益是否同样不迁移。
- 产出:用处一说的那张图(现在多了小 σ 端的形状),σ\* 的数值,以及 RandOpt σ 网格中有几档落在噪声等效区的判定。
- 结果怎么读:
  - 小 σ 端出现平台(漂移不随 σ 下降,高度与 permutation 线相当,直到噪声全部被量化吞掉、跳到逐比特零),则精度下限成立:σ 低于 σ\* 时,测到的扰动效果主要是数值噪声,不反映扰动本身;
  - 小 σ 端连续降到零,说明高斯扰动的数值噪声比 permutation 的小,精度下限只适用于更小的效果量级,如实缩小结论的适用范围;
  - 若 RandOpt 网格最低一两档的漂移不超过 permutation 漂移,这些档位的候选在 logits 上与函数完全相同的模型不可区分,挑出的提升应对照 Part 3 的假象曲线解读;
  - 行为级若发现低 σ 高斯候选翻的题和 permutation 翻的题是同一批,说明不稳定题集是模型自身的属性(低 margin 决策点),与扰动来源无关。
- 成本:logits 级约 5 小时(30 次引擎加载);行为级约 2.5 小时(10 次 GSM8K);权重级 0。

### Part 7:ZO 探针的噪声下界(Part 6 的顺带产出,几乎零额外成本)

- 问题:ZO 微调在 bf16 下用 ε=1e-4~1e-3 的试探差值估方向,这个差值信号会不会低于纯数值噪声能造成的 loss 变化?
- 操作:在 Part 6 的 logits 级测量中,顺带记录每个配置在 32 个 prompt 上的平均 NLL(loss)。这样得到两个分布:permutation 各 seed 的 loss 相对原模型的变化(纯数值噪声下界),以及各档 σ 高斯扰动的 loss 变化。比较 ε=1e-4 和 1e-3 对应档位的 loss 变化量与下界的比值。
- 产出:一张"扰动大小对 |Δloss|"图,标出数值噪声下界横线。
- 结果怎么读:若某档 ε 的 |Δloss| 与下界同量级,该设置下 bf16 的 ZO 探针信噪比接近 1,方向估计主要在拟合浮点噪声,这对 ZO 社区是直接可操作的警示(需要更大的 ε 或更高精度的 loss 累加)。若远高于下界,说明 ZO 的常用设置是安全的,同样值得写清楚。
- 成本:并入 Part 6,额外 GPU 时间约为 0。

## 5. 明确不做的事(边界)

1. 不解释 Instruct 和 Base 方向相反的深层原因(是训练方式还是任务分布造成的)。那需要更多模型、更多执行栈,超出本项目。Part 4 只判定方向是不是回归均值,不追问更深。
2. 不做温度采样下的实验。现有结论都在确定性 greedy 解码下,采样分布如何变化留作已声明的开放问题。
3. 不扩展到其他模型、GPU 或框架。所有结论的适用范围按 [`current_findings.md`](../reports/current_findings.md) 的口径声明边界。
4. 不再为 PyTorch 单 token(GEMV)例外开新实验。已定性:微噪声,无害,不逐比特。记录在案即可。
5. 不复现 RandOpt 的完整流程(几千个候选、support-set 打分、top-k 聚合)。我们只测扰动本身的物理量级(σ 扫描 30 个点,加可选的 10 次 GSM8K),把"选择增益是否真实"的判定留给 Part 3 的对照曲线,不替它跑搜索。

## 6. 最终产出

跑完后写一份收尾文档,把四个实验串成完整故事:

1. 存在严格零扰动的置换类(块内),从矩阵乘到 benchmark 全链条逐比特验证;
2. 跨块置换产生的漂移被模型的 margin 冗余逐级吸收,只在长生成任务留下 ≤1pp 波动,每一级有实测数字;
3. 该波动无方向、不可利用:挑 seed 的增益不迁移,方向差异是回归均值;
4. 一张扰动谱图(核心图):从半径 0 到 σ=1e-2 的完整刻度,标出逐比特区、数值噪声区、量化吞噬区、σ\*、RandOpt/MeZO/FZOO 的工作点。这张图就是本项目和随机扰动搜索方向之间的接口;
5. 两件可复用的工具:功能等价候选池上的假象增益曲线(供邻域挑模型的方法校零),以及 ZO 探针噪声下界(bf16 下 ε 的安全线)。
