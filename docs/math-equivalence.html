Mathematical intuition · exact symmetry

把神经元换个座位，为什么函数没有变？

因为这里做的不是“把功能搬错位置”，而是给 FFN 的内部临时坐标重新编号：入口与出口坐标都不动，三条配套线路一起改名，离开中间空间前再翻回原字典。

对象 SwiGLU FFN 模型 Qwen3-4B 中间维 m = 9728 最后核对 2026-07-13
精确实数函数完全相同
BF16 执行张量按置换类别判断
最终任务行为块内零差，跨块可变
最终轮补充边界

“数学等价”不自动保证任意 BF16 执行都逐 bit 相同，但也不是所有置换都会漂移。在当前 vLLM batch-invariant 栈中，只在原对齐 8 通道块内重排已经从 logits 到六项 benchmark 逐题验证为零差；跨块重排才进入 backend 相关的非零漂移范围。

01
One-sentence model

换编号，不换功能

先暂时忘掉矩阵。把 FFN 想成若干个并行的小单元，每个单元算出一份贡献，最后把所有贡献加起来。

最重要的一句话

联合 permutation 只是把 1, 2, …, m 这套内部编号换成另一套编号。只要每个单元的“开关、内容、写回方向”始终绑在一起，最终只是把同一批求和项换了枚举顺序。

x模型隐藏状态
入口坐标不变
→
P进入中间空间后
给坐标换名
→
P−1离开前把名字
翻回原字典
→
y输出隐藏状态
出口坐标不变

四个求和项的玩具例子：右侧只改变顺序，项本身没有拆散。

原编号 · 1, 2, 3, 4
1v₁ · a₁(x)bundle A
2v₂ · a₂(x)bundle B
3v₃ · a₃(x)bundle C
4v₄ · a₄(x)bundle D
y = A + B + C + D
π →
新编号 · π = (3, 1, 4, 2)
3v₃ · a₃(x)bundle C
1v₁ · a₁(x)bundle A
4v₄ · a₄(x)bundle D
2v₂ · a₂(x)bundle B
y′ = C + A + D + B = y
02
What is one neuron?

这里的“一个神经元”，其实是三件套

SwiGLU 有 gate 与 up 两条输入分支，再由 down 写回 hidden space。因此只看一张权重矩阵的某一行，会漏掉这个内部坐标真正的完整定义。

SwiGLU
f(x) = Wd[ SiLU(Wgx) ⊙ (Wux) ]
gate · 第 k 行

开关

SiLU(wg,kTx) 决定这一内部特征被放行多少。

up · 第 k 行

内容

wu,kTx 提供这一内部特征携带的数值。

down · 第 k 列

写回方向

vk 把标量贡献投回原来的 hidden 坐标。

按神经元展开
f(x) = Σk=1m vk · SiLU(wg,kTx) · (wu,kTx)
因此真正应该一起移动的是 bundle

第 k 个内部坐标是三元组 (wg,k, wu,k, vk)。三件套一起移动只是重排求和项；只移动其中一件，就把别人的开关、内容或写回方向配在了一起。

输入 / 输出维
d = 2560。这是 Transformer residual stream 的坐标，置换前后都不改变。
中间维
m = 9728。这里没有被外部直接观察的固定编号，所以允许重新命名。
权重形状
Wg, Wu ∈ ℝm×d；Wd ∈ ℝd×m。
bias
当前 Qwen3 三个 projection 都是 bias=False。若存在 gate/up bias，也必须随对应行一起置换。
03
Two exact proofs

同一件事的两种严格证明

逐神经元视角最直观；矩阵视角最适合检查实现。二者证明的是同一个命题：参数不同，但对所有输入实现的精确函数相同。

证明 A · 求和项

重排有限和

设 π 是 {1,…,m} 的双射。联动置换后，每一个 bundle 都完整保留，只是第 k 项变为原来的第 π(k) 项。

Σk bπ(k) = Σj bj

因为 π 不丢项也不重复项，所以精确有限和相同。

证明 B · 坐标变换

中间改名，出口撤销

令 permutation matrix 为 P。gate/up 在输出行上左乘 P；down 在输入列上右乘 PT。

θP = (PWg, PWu, WdPT)

由于 PTP = I，中间坐标变换在出口被准确撤销。

矩阵推导
h′=SiLU(PWgx) ⊙ (PWux)定义
=P·SiLU(Wgx) ⊙ P·(Wux)SiLU 逐坐标
=P[SiLU(Wgx) ⊙ (Wux)]两支使用同一个 P
=Ph
y′=WdPTh′ = WdPTPh = Wdh = y
为什么 SiLU 可以“穿过” P？

SiLU 对每个坐标独立作用。先重排再逐元素计算，与先逐元素计算再重排完全一样：φ(Pz)=Pφ(z)。

为什么两个分支必须同一个 P？

Hadamard 乘法按位置配对。只有 (Pa)⊙(Pb)=P(a⊙b)；若两边用不同置换，配对关系就变了。

更抽象地说：这是参数空间里的 permutation orbit

定义 θ ~ θ′ ⇔ fθ(x)=fθ′(x), ∀x。那么所有 TP(θ) 都与 θ 属于同一个等价类。每层的中间坐标互相独立，36 层整体可分别选择 Pℓ，对应的离散对称群可写作 Sm₁ × … × Sm₃₆。

04
Indexing convention

矩阵里是 PT，代码里为何三处都写 perm？

这是实现中最容易写反的地方。关键在于先固定一个约定：本项目把 z[..., perm] 定义为置换后的向量。

# 约定：z_perm = z[..., perm]
gate.weight = gate.weight[perm, :]   # 行重排：P W_g
up.weight   = up.weight[perm, :]     # 行重排：P W_u
down.weight = down.weight[:, perm]   # 列重排：W_d P^T

# 恢复时才使用 inverse
inv_perm = torch.argsort(perm)
“同一个索引写法”不等于“同一个矩阵乘法方向”

gate/up[perm, :] 是重排行，对应左乘 P；down[:, perm] 是重排列，对应右乘 PT。数组索引看起来相同，是因为行与列所处的张量位置不同。

实现方向检查表
对象轴代码矩阵记号语义
gate输出行[perm, :]P Wg给开关换编号
up输出行[perm, :]P Wu给内容换同一编号
down输入列[:, perm]WdPT按新编号读取，再写回原空间
方向单测

一定用非自逆 permutation

reverse、half-swap、单次交换都是自逆的，写成 perm 或 argsort(perm) 看不出差别。随机非对合置换才能抓住方向 bug。

canonical-down

先逆排，再走原 down

把 h′ 用 inverse 恢复成原坐标，再送进原 down。项目中 120/120 case 与 baseline 逐 bit 相同，直接证明公式与索引都没错。

05
Exact math ≠ finite arithmetic

数学等价，为何 BF16 仍会出现差异？

精确实数代数只关心“有哪些求和项”；GPU kernel 还关心这些项如何分 tile、如何归约、最后落到哪个 BF16 格点。置换不改精确值，却会改变物理执行路径。

函数等价
严格成立 对任意实数输入，fθ(x)=fθP(x)。
坐标等价
逐 bit 通过 gate / up / product 在重排坐标后 120/120 完全相同。
BF16 张量
近似 native down_proj 的物理布局改变，低位舍入可落到相邻格点。
任务行为
经验判断 大多数 token 不变；低 margin 分叉与长生成会放大微小差异。

误差第一次出现在哪里？

gate/up 做的是“整行搬家”：每个点积内部的输入顺序没有变，输出只是换了存放位置；SiLU 与逐元素乘也只跟着位置走。因此直到 h′=Ph 都可逐 bit 对齐。第一次可见差异出现在 down_proj：同一批乘积被放入了不同的物理列与 kernel 分组。

纯 FP32 归约顺序
2.5e−7

相对误差，极小。真实 kernel 的 FP32 accumulation 本身非常准确。

单次 BF16 GEMM
2.325e−3

对 FP64 参考的固有误差，接近 ulp/√12 = 2.255e−3。

两种布局之差
3.09–3.22e−3

接近 √2 × ulp/√12 = 3.189e−3。

更精确的机制图景

置换改变 tile / reduction tree 后，FP32 内部结果只在很低位略有不同；写回 BF16 时，靠近量化边界的元素可能分别落到相邻格点。就本模型、输入与 kernel 的实测而言，置换差异近似于把 BF16 输出量化误差重新抽样一次。它是实验观察到的定量闭环，不应外推成对所有硬件与 kernel 都成立的普适上界。

为什么早期的“BF16 链式加法吃掉小项”还不够精确？

它正确说明了有限精度加法不满足实数域的结合律，但真实 GPU GEMM 并不是把 9728 项直接用 BF16 一项项累加。后续分解实验测得 FP32 累加顺序效应只有约 2.5e−7；主导可见漂移的是结果写回 BF16 格点时的边界翻转。因此正式解释应把“归约布局变化”与“BF16 输出量化”连起来。

不要把“逐 bit 不同”误判成“公式不成立”

FP64 valid-triplet 的误差约 1.4e−14；真正错误的单矩阵或错方向置换，在同样设置下是 O(1) 的函数破坏。BF16 valid 的小漂移与错误变换之间有 235–317 倍的清晰分离。

06
Counterexamples

哪些说法并不成立

“存在联合对称性”是一条非常具体的命题。把它扩大成“每张矩阵都能随便换”或“任何两个模型都等价”，都会越过证据边界。

×

单独置换 gate / up

开关与内容的坐标错配。Hadamard 乘法会把原本不属于同一 bundle 的两项乘在一起。

×

单独置换 down

中间激活本身没换编号，down 却按另一套字典读取，于是贡献被写回错误方向。

×

三个独立 permutation

三件套被拆散。只有 gate 与 up 共用 P、down 使用匹配的 PT 才成立。

×

两个 checkpoint 自动等价

合法置换只保证每个 checkpoint 自己的精确函数不变，不保证独立训练模型之间存在完美一一匹配。

08
Glossary & provenance

术语与证据入口

所有结论都对应仓库内可追溯的计划、实现、原始结果或聚合报告。

permutation / permutation matrix

permutation 是有限索引集合上的双射；permutation matrix P 每行每列恰有一个 1。它是正交矩阵，所以 P−1=PT。

rel_l2

‖a−b‖₂ / max(‖a‖₂, ε)。它衡量相对于参考张量整体尺度的差异，不能直接解释成“准确率下降百分比”。

bitwise equal / functionally equivalent

bitwise equal 要求存储的每一位都相同；functionally equivalent 要求对所有输入的精确函数值相同。有限精度不同执行路径可以满足后者而不满足前者。
