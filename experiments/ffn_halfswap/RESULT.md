# 实验二结论：FFN 前后半块交换（all-36 half-swap）

> **阅读说明：** 本文中“统计上不可区分”指描述性统计落在相同观测量级，并未执行正式的统计等价性检验。跨实验总结统一采用“同一观测量级”的谨慎表述。

> 执行日期：2026-07-10；环境同实验一（RTX 4090，BF16，PyTorch 2.13.0+cu130）
> 置换：每层 `perm = [4864..9727, 0..4863]`（m=9728 的前半 ↔ 后半，自逆），
> 联动作用于 gate/up 行 + down 列，36 层全部施加
> 输入：实验一的 32 条固定 prompt（逐 bit 相同的 input_ids）
> 原始数据：`results/halfswap.jsonl`、`results/manifest.json`

## 0. 一句话结论

**全 36 层前后半块交换后，模型函数不变：温度 0 下两种条件各自 8 次重复完全逐 token 一致（零方差）；与 baseline 相比 32 条 prompt 中 28 条的 64-token 回答逐 token 完全相同，4 条在近并列分叉处发生同义改写；logits 漂移量级（rel_l2 ≈ 2.1e-2）与实验一随机置换完全一致，即 half-swap 没有任何超出"BF16 归约噪声"之外的影响。**

## 1. 温度 0 下的重复采样（每条件 8 次）

| 检查 | baseline | half-swap |
|---|---|---|
| forward logits 重复 2 次 bitwise 一致 | 32/32 | 32/32 |
| 贪心生成独立重复 8 次逐 token 一致 | 32/32 | 32/32 |

温度 0 是确定性策略：**条件内不存在任何随机性**（256 次生成、128 次 forward 无一漂移）。因此下面所有 baseline vs half-swap 的差异都可以完全归因于置换本身，与采样无关。这也说明对确定性策略"多次采样"不会产生新信息——变化只发生在条件之间，不发生在条件之内。

## 2. logits 对比（half-swap vs baseline）

| 指标 | 值 | 实验一 all-36 随机置换（参照） |
|---|---:|---:|
| 全 token rel_l2 median / max | 2.13e-2 / 2.80e-2 | 2.0–2.2e-2 / 3.1e-2 |
| `mean\|Δ\|` / `max\|Δ\|`（median） | 0.061 / 0.76 | 0.062 / 0.81 |
| cosine（最小值） | 0.99948 | ≥0.99933 |
| top-1 agreement | 98.25%（22/1255 翻转） | 97.8–98.5% |
| 翻转处 baseline margin median | 0.031（全体 median 1.875） | 0.031–0.125 |
| last-token top-1 一致 | 32/32 | 31–32/32 |

half-swap 的数值特征与随机置换**统计上不可区分**——块交换同样打乱 down GEMM 的归约分组，产生同量级舍入噪声；置换的"形状"（块交换 vs 随机打乱）不影响结果，这与实验一"误差只与归约分组是否改变有关"的机制定位一致。

## 3. 最终回答对比（温度 0，64 tokens）

- **28/32 条逐 token 完全一致**（包括全部数学、翻译、代码类 prompt 的答案本身）。
- 4 条发生分叉：prompt 6（@token 63）、16（@55）、29（@45）三条在**接近生成末尾**处才分叉（可见前缀完全相同）；prompt 30（@11）较早分叉，为同义改写：

  > base：`春节：中国最重要的传统节日，象征新年的开始，家人团聚、庆祝丰收。`
  > swap：`春节：中国最重要的传统节日，庆祝新年的开始，有贴春联、放鞭炮和家庭团聚的习俗。`

- 分叉集中在 margin<0.13 的近并列 token 上，事实性内容无一改变；prompt 30 恰好也是实验一 all-36 随机置换中最常分叉的 prompt——分叉位置由"baseline 中本来就近并列的分叉点"决定，而非由置换方式决定。

## 4. 校验

- 置换为自逆（`argsort(perm) == perm`，已断言）；实验后 inverse 还原，36 层 × 3 矩阵与 CPU master copy `torch.equal` 全部通过，全 MLP 权重 SHA-256 首尾一致。
- 未落盘任何 permuted checkpoint。

## 5. 对用户问题的直接回答

1. **换了之后最终会影响输出结果吗？** 数学上不会（函数不变）；工程上 BF16 logits 有 ~2% 的相对舍入漂移，导致 32 条中 4 条回答在近并列分叉处换了说法（内容等价）。若在 FP64 下做同样的事，输出会严格不变（实验一 A 层已证）。
2. **温度 0 多次采样的变化如何？** 条件内完全零变化（8/8 一致）；全部变化都出现在 baseline 与 half-swap 两个条件之间，且量级、位置与随机置换的情形完全相同。
