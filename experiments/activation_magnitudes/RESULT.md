# 实验三报告：Qwen3-4B 的 activation / 运算量级画像

> **后续边界说明（2026-07-13）：** 本文中的“天花板”描述来自当前模型、三层探针与当时的 PyTorch BF16 kernel。后续实验发现 vLLM 的平台形态不同，PyTorch 单行 GEMV 也构成例外。因此，下文较强的“硬上界”措辞应按当前实测域理解，不应外推为所有硬件、形状或 kernel 的普适定理。更新后的边界见 [`../../docs/reports/current_findings.md`](../../docs/reports/current_findings.md)。

> 日期：2026-07-11；环境：RTX 4090，BF16，PyTorch 2.13.0+cu130
> 输入：从 `/nvme0/if/llm-brewing/bench` 的 9 个基准（mmlu / mmlu_pro / mmlu_redux /
> ceval / cmmlu / gsm8k / math500 / bbh / cruxeval）各随机采 16 条（seed 42），
> 共 **144 条 prompt、16132 个 token**，chat template 格式，截断至 512 token
> 原始数据：`results/magnitudes.json`、`results/down_gemm_decomposition.json`、
> `results/prompts_used.json`；复现：`python probe_magnitudes.py`

## 0. 要点

1. 模型内部的"典型数字"横跨 6 个数量级：embedding ~0.02，MLP 内部激活 ~0.03–4，
   残差流 RMS ~32 但**个别通道冲到 16384**，logits ~4。BF16 对所有这些量级的相对
   分辨率恒定为 1/128 ≈ 0.8%（刻度随量级等比变粗）。
2. down_proj 求和存在 **30–70 倍的正负对消**：9728 个乘积的绝对值之和是最终结果的
   几十倍，单个最大乘积就有结果的 0.3–0.6 倍——"大进大出、几乎抵消"。
3. **漂移的定量闭环**（本报告最重要的结果）：单次 BF16 down-GEMM 的固有误差
   = 2.32e-3，几乎恰好等于 BF16 输出量化的理论下限 ulp/√12 = 2.26e-3；置换重排后
   两种顺序的输出差 = 3.09–3.22e-3，几乎恰好等于 √2 × 下限 = 3.19e-3。
   **即：置换漂移 = 把输出量化噪声独立地重抽了一次，它的天花板就是模型自身的
   BF16 量化噪声，不构成新的误差类别。**

## 1. 全模型量级地图

各流在全部 16132 个 token 上的统计（RMS 为 token 加权；完整 36 层数据见
`results/magnitudes.json`）：

| 位置 | RMS | abs p99 | abs max | BF16 刻度@RMS | 说明 |
|---|---:|---:|---:|---:|---|
| embed 输出 | 0.020 | 0.052 | 0.16 | ~1.6e-4 | 极小，起点 |
| 残差流（L8–L33 稳态） | ~32 | — | **16384** | 0.25（outlier 处 **128**） | 少数"massive activation"通道撑起 max |
| MLP 输入（layernorm 后） | 0.11–1.7 | — | — | ~0.004 | 归一化把量级拉回 O(1) |
| gate 输出（SiLU 前） | 0.2–9.7 | — | — | ~0.03 | L1–L2 出现 8–10 的尖峰 |
| up 输出 | 0.18–3.1 | — | — | ~0.01 | |
| h = silu(g)⊙u（down 输入） | 0.03–3.7 | — | **1848**（L34） | ~0.01 | 深层出现大 outlier |
| MLP 输出（down 后） | 0.09–15.6 | — | **7008**（L34） | ~0.06 | L34/35 急剧变大 |
| final norm 输出 | 2.74 | 7.5 | 153 | 0.02 | |
| **logits** | **3.77** | 10.6 | 52.8 | 0.03 | top1-top2 margin 中位数 **2.56** |

逐层看三个规律（代表层摘录，完整见 JSON）：

| layer | 残差流 RMS / max | h RMS / max | mlp_out RMS / max |
|---:|---|---|---|
| 0 | 0.19 / 9 | 0.042 / 12 | 0.14 / 7 |
| 1 | 0.34 / 66 | 0.087 / 72 | 0.24 / 61 |
| 8 | 31.4 / 16192 | 0.130 / 28 | 0.32 / 13 |
| 17 | 31.9 / 16384 | 0.134 / 17 | 0.31 / 13 |

| 32 | 32.7 / 16384 | 0.709 / 170 | 1.78 / 61 |
| 34 | 29.1 / 13632 | 1.420 / **1848** | 6.68 / **7008** |
| 35 | 17.9 / 7040 | 3.733 / 952 | 15.6 / 6624 |

- **残差流在 L8 前指数增长后进入 ~32 的稳态平台**，其中 max=16384 来自少数
  超大通道（文献中的 massive activations / attention-sink 现象），BF16 在
  16384 处的刻度是 128——但相对分辨率仍是 0.8%，且 layernorm 在进入每个子模块
  前会把量级归一，所以粗刻度不直接伤害计算；
- **MLP 内部激活是 O(0.1–1) 的"细刻度区"**（刻度 ~1e-3），大数值只在深层
  （L34/35）的少数元素上出现；
- 权重本身很小且均匀：gate/up/down 的 RMS ≈ 0.023，abs max ≤ 0.86。

## 2. down-GEMM 内部：一笔"大进大出"的账

对 L0 / L17 / L35 各取 64 个真实 token 的 h，逐项分析
`y_i = Σ_k W[i,k]·h[k]`（9728 项）：

| layer | 单项 RMS | Σ\|项\| (p50) | \|y\| (p50) | **对消比 Σ\|项\|/\|y\|** | max\|项\|/\|y\| |
|---:|---:|---:|---:|---:|---:|
| 0 | 0.0006 | 1.36 | 0.043 | **32×**（p99: 1785×） | 0.61 |
| 17 | 0.0030 | 13.8 | 0.19 | **68×**（p99: 3750×） | 0.35 |
| 35 | 0.0550 | 62.2 | 2.31 | **28×**（p99: 1454×） | 0.45 |

即：求和过程中流经的数值比最终结果大 1–3 个数量级，最终结果靠大规模正负
对消得到。这正是"求和顺序敏感"的高危形态——不过下一节会看到，fp32 累加器
把这个风险几乎完全吸收了。

## 3. 漂移的定量闭环：三层分解

用同一批真实 h（bf16）和 down 权重（bf16），把"重排求和顺序"的效应逐级拆开
（L17，3 个随机置换 seed；L0/L35 同量级，见 `results/down_gemm_decomposition.json`）：

| 对比 | rel_l2 | 解读 |
|---|---:|---|
| fp32 GEMM vs fp64 精确 | 1.9e-7 | fp32 累加自身极准（对消比只把它放大到 ~3×eps_fp32） |
| fp32 GEMM 原顺序 vs 置换顺序 | **2.5e-7** | **纯累加顺序效应：可忽略** |
| fp32 结果各自舍到 bf16 后对比 | 0.8–1.7e-5 | 若 kernel 是"一条 fp32 链算到底再舍入"，漂移只有这么大 |
| bf16 GEMM vs fp64 精确 | **2.325e-3** | 单次 bf16 GEMM 固有误差；理论量化下限 ulp/√12 = **2.255e-3** |
| bf16 GEMM 原顺序 vs 置换顺序 | **3.09–3.22e-3** | 实测漂移；理论 √2×下限 = **3.189e-3** |

三个结论：

1. **单次 bf16 GEMM 的误差几乎 100% 是"把 fp32 累加结果写回 BF16 格点"的输出量化**
   （2.325e-3 vs 理论 2.255e-3，多出的部分即 kernel 内部结构的贡献）；
2. **置换后的漂移 = √2 × 单次量化噪声**（3.19e-3 理论 vs 3.09–3.22e-3 实测）：
   重排让 kernel 内部的 tile / 归约树分组改变，处在格点边界附近的输出元素翻到
   相邻格点，两种顺序的量化误差变成近乎**独立的两次抽样**；
3. 因此**置换漂移存在硬天花板：它等价于把模型自身的 BF16 量化噪声重抽一次**，
   永远不会比"模型本来就承受的量化"更糟。Stage B 实测的单层 4.7e-3（跨 3 层
   × 5 种输入 × 5 seed）与这里的 3.1–3.2e-3 同量级，略高来自输入分布差异。

> **对 [`DRIFT_ORIGIN.md`](../ffn_permutation/DRIFT_ORIGIN.md) 的定量细化**：
> 该文用"小项被大余额的粗刻度吃掉"解释不可结合性，机制本身正确（BF16 链式
> 加法确实如此），但真实 GPU kernel 用 fp32 累加，"累加内部吸收"的效应只有
> 1e-7 量级；可见漂移的主通道是**输出量化的边界翻转**。刻度/格点的图景不变，
> 只是"舍入发生的位置"从累加过程移到了写回输出那一步。

## 4. 与端到端现象的连接

- logits RMS 3.77、top1-top2 margin 中位数 **2.56**；全 36 层置换后 logits 相对
  漂移 ~2e-2（实验一/二），折成绝对量 ~0.06（mean）——距离典型 margin 还差
  40 倍，所以 98% 的 token 不翻转，翻转只发生在 margin<0.13 的近并列处；
- 单层量化噪声 ~2.3e-3 → 经 36 层传播饱和到 logits 的 ~2e-2：放大约一个数量级，
  与实验一的逐层曲线一致；
- 残差流的 16384 级 outlier 通道解释了为什么误差在中段"被吸收"：相对于稳态
  RMS 32 的残差流，单层 MLP 注入的绝对扰动（~0.3×2.3e-3≈7e-4）微不足道。

## 5. 工程含义

1. **BF16 下任何会改变 GEMM 归约分组的操作**（permutation、张量并行切分方式、
   kernel 版本/tile 尺寸变化、batch 形状变化）都会触发同一类 ~ulp/√12 的输出
   量化重抽样——做逐 bit 回归对比时必须预期这个下限；
2. 若下游需要压掉这个漂移：用 fp32 权重跑对比（漂移降到 1e-5 级）或在 fp32
   中做置换验证（本轮 canonical 路径已证可逐 bit 复原）；
3. massive activations（16384 级通道）对 BF16 的相对分辨率不构成额外风险，
   但任何想动残差流本身的操作（如通道级 merge/剪枝）需要单独评估这些通道。
