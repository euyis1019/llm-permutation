## 0. 一页总览

**问题**:对 FFN 做联动置换在数学上严格等价,但 BF16 下有微小数值漂移。如果必须置换一部分 neurons,怎么组织置换才能让代价最小?

**目前的答案(复审后)**:单层漂移不是连续量,是三档阶跃——

| 档位 | 条件 | 单层漂移 |
|---|---|---|
| 免费 | 置换保持"对齐 8-块"成员关系 | **恰好为 0(逐比特)** |
| 中间 | 只在对齐 16~32 窗内混合 | ~3×10⁻⁵ |
| 饱和 | 实质性跨块(其余几乎一切) | ~3×10⁻³,天花板,与置换多少无关 |

**判据是对齐块,不是移动距离**:偶对齐交换模式 (0,1)(2,3)… 逐比特免费;奇对齐交换模式 (1,2)(3,4)… 位移同样是 1,在 torch 下却直达天花板。

**实质跨块近似开关,不是斜坡**(最常见的误读,见 §6.1):单层内"跨块的 neuron 越少漂移越小"**不成立**——散置 5% 与 50% 的漂移一模一样。跨块数量在单层内不是连续旋钮;真正连续的剂量轴在**层间**(多少个层非免费、在多深)。

**因此**:能设计成块内重排 ⇒ 代价严格为零;必须跨块 ⇒ **跨块数量本身不能预测代价**,虽存在一条稀窄且数据依赖的中间档,但一旦形成实质性跨块通常就迅速饱和。此后更可靠的杠杆是"动哪些层"。

**确认状态(v1.1 阶段 1b 已执行,见 §8)**:免费类已在全新 seed 上正式确认于 **torch GEMM(T≥2)、vLLM batch-invariant prefill、vLLM batch-invariant decode** 三个组合(378/378 逐比特,无一例外);**例外是 torch 的 T=1 GEMV 路径**——该 kernel 的归约结构没有连续 8-块,任何置换(含块内重排)都有 ≤6e-5 的偶发微噪声,同时也没有天花板(连全局随机都只有 sub 级)。部署栈(vLLM)上免费类完整成立;用 HF transformers 逐 token 解码则"逐比特为零"退化为"≤1e-4 的微噪声"。

## 1. 三个实验的关系

本目录是系列的第三个实验:

| 实验 | 问题 | 一句话结论 |
|---|---|---|
| `experiments/ffn_permutation/`(上游) | 联动置换在 logits/激活层有多大漂移? | 漂移真实存在,rel_l2 ~2e-2(全 36 层),100% 来自 down_proj 的 BF16 归约顺序变化;越早的层注入放大越多 |

| `experiments/permutation_min_cost/`(本实验) | 怎么组织置换让代价最小? | 见 §0;正在从"失败的连续定律"修正为"三档定律" |

### 1.1 `rel_l2` / `REL` 是什么

这里的 `rel` 是 **relative(相对)** 的缩写。文中的 `rel_l2` 是置换后输出相对于原始输出的相对 L2 误差:

$$
\operatorname{rel\_l2}
= \frac{\left\lVert y_{\mathrm{perm}}-y_{\mathrm{base}}\right\rVert_2}
       {\left\lVert y_{\mathrm{base}}\right\rVert_2}
$$

它把整个输出张量展平后计算整体距离,是无量纲的。`rel_l2=0` 表示数值完全相同(但只有同时满足 `bitwise_equal=true` 才明确表示原始张量逐比特相同);`3e-3` 表示两个完整张量的 L2 距离约为基准张量 L2 范数的 0.3%。这**不等于**"每个元素都变化了 0.3%",也不是逐元素相对误差的平均值。实现见 `scripts/permutation.py::diff_metrics`。

### 1.2 怎么理解"小但真实"和"归约扰动 × 传播深度"

先澄清量纲:`pp` 是 **percentage point(百分点)**,不是相对百分比。例如准确率从 50.0% 变为 49.2%,差值是 −0.8pp(相对下降则是 1.6%)。表中 `0.5–0.8pp` 指六项 benchmark 等权平均后的六任务平均分;单个敏感的生成任务可偏移约 1–2.3pp。

- **为什么多选题几乎免疫**:MMLU/C-Eval/CMMLU 最终通常只需做一次单 token 选项决策。BF16 漂移虽然让许多 logits 的末位发生变化,但只要正确选项与次选项之间的 margin 足够大,最大项就不会变。实验测试了 20 组随机置换（由 20 个不同随机种子各生成一套置换），没有一组超过 ±1pp,逐题 correctness 一致率 ≥99%。
- **为什么长生成更敏感**:GSM8K 和代码生成需要连续产生很多 token。某一步若遇到两个候选 token 很接近的低-margin 分叉,微小漂移就可能改变所选 token;这个 token 又成为后续输入,使后面的整条生成路径改变。最终答案或代码测试结果因此比单 token 多选更容易翻转。
- **"方向随模型固定"是什么意思**:汇总这 20 组随机置换后,Qwen3-4B-Instruct 的六任务平均分为 −0.79pp,Qwen3-4B-Base 为 +0.49pp。同一模型内偏移呈稳定倾向,不像以 0 为中心的纯随机抖动;但两个模型方向相反,所以**不存在"置换总会变好"或"总会变差"的普适方向**。
- **"归约扰动 × 传播深度"是什么意思**:第一项是置换是否改变 `down_proj` GEMM 的 BF16 分块归约路径——保持对齐 8-块成员关系时扰动为 0,跨块时才产生漂移;第二项是漂移注入后还要穿过多少下游层——越早的层注入,传播和放大的路径越长。只动 L0 已产生全 36 层约 75% 的 benchmark 影响,只动 L35 则几乎没有影响。这里的 `×` 是**机制上的概念表达**,表示结果同时受两项制约,不是已经拟合出的精确乘法公式。对长生成任务,还会额外叠加上述逐 token 的自回归分叉效应。

一句话说:置换先在某个 `down_proj` 制造一个可能为零、也可能达到 BF16 天花板的微小扰动;它越早进入网络、生成链越长,就越有机会在低-margin 决策点改变离散答案。不过即使最敏感的配置,六任务平均分变化仍只有约 0.5–0.8pp,所以称为"小但真实"。

## 2. 基础概念:为什么数学等价,又为什么有漂移

### 2.1 联动置换

Qwen3-4B 的 FFN 是 SwiGLU 结构:

```
y = W_down · ( silu(W_gate · x) ⊙ (W_up · x) )
```

中间维度(neurons)有 M=9728 个。取任意置换 π,同时做三件事:

- `W_gate`、`W_up` 的**行**按 π 重排;
- `W_down` 的**列**按同一个 π 重排。

中间激活 h 的每个分量只是换了位置,和它配对的 down_proj 列也跟着换到同一位置,所以**每个乘积项 W_down[:,i]·h_i 的配对关系严格不变**,数学上 y 完全不变。这就是"等价变换"。

### 2.2 漂移从哪来

浮点加法不满足结合律:`(a+b)+c ≠ a+(b+c)`(最后一位舍入可能不同)。置换不改变"加哪些数",但改变这 9728 个乘积项**的排列位置**,从而改变 GPU 求和时的**加法顺序**。上游实验已逐比特证明:整个模型的漂移 100% 来自 down_proj 这一个矩阵乘的归约顺序变化,其他算子(silu、逐元素乘等)对置换严格不敏感。

### 2.3 GPU 求和是分块的(理解三档结构的钥匙)

一个输出元素 = 长度 9728 的点积。GPU kernel 不是从左到右累加,而是:

```
9728 个乘积项
 └─ 切成对齐小块(实测宽 8):[0..7] [8..15] [16..23] ...
     └─ 块内:tensor core 硬件一次算完,块内顺序无影响
     └─ 块间:部分和逐级累加,有顺序、有中间舍入
```

- 置换只在块**内**动 → 每块拿到同一批项 → 部分和逐比特不变 → 输出逐比特不变(**免费档**)。
- 有项被移**出**自己的块 → 相关块的部分和变了 → 中间舍入路径变了 → 最终 FP32 结果有微小差别。输出还要舍到 BF16(只有 8 位尾数,格距 ~0.4%),这个微小差别使每个输出元素"向上舍还是向下舍"近似掷硬币 → 实测 35–47% 的输出元素恰好翻 1 个最低位,rel_l2 锁死在 ~3e-3 ≈ BF16 格距(**饱和档**)。
- 只挪到紧邻块(16~32 窗内) → 部分和变化很小,只有 ~0.01% 的元素被推过舍入临界点(**中间档**)。

**饱和档是天花板不是累积量**:误差大小由输出 BF16 格距决定,和"扰动了多少项"无关。散置 5% = 散置 50% = 全局随机 = 反转,单层漂移全部 ~3e-3。除稀窄、数据依赖的过渡区外,实质性跨块近似一个开关:触发后就到顶。

## 3. 本实验最初的设计(EXPERIMENT_PLAN.md,现已部分作废)

### 3.1 思路

当时的假设:存在一个连续规律"漂移 = f(置换的几何特征)"。若找到 f,就能预先算出任何置换的代价,进而构造最小代价置换。

### 3.2 四个阶段

1. **阶段一(单层定律)**:8 个置换族 × 5 个 seed ≈ 147 个置换实例——窗口乱序(F1,宽度 2~9728)、连续块内乱序(F2)、全局散置(F3)、相邻成对交换(F4)、固定距离成对交换(F5,距离 1~4096)、大块对调(F6)、全局随机(F7)、反转/恒等(F8)。每个实例在真实权重+真实激活上只算 down_proj 一个矩阵乘,测漂移;同时算 6 个几何量(移动比例、平均/最大/总位移、跨块计数、逆序数),回归找预测指标。
2. **阶段二**:单层预测 → 全模型 logits 的穿透性验证。
3. **阶段三**:用规律构造最小代价 30% 置换,与朴素/最差三臂对比跑 benchmark。
4. **阶段四**:instruct 模型方向确认。

### 3.3 预注册纪律(为什么"失败要停机")

文档是给另一个 agent(codex)长程自主执行的,所以把验收标准和停机规则写死:

- **S1a**:≥200 个非零漂移点,且至少一个几何指标在真实激活上 R²≥0.8 或秩相关 ρ≥0.9(三层都要);
- **S1b**:F5 族漂移随交换距离单调,每层 ρ≥0.9;
- **S1c**:相邻成对交换(K=30%)的漂移中位数 < 全局散置的 1/3;
- **不达标 → 写失败报告、立即停机;禁止调阈值、换 seed、增删族**(防止执行方把标准改到能过)。

## 5. 复审:为什么"失败"其实是出题错了

复审(reviewer 角色)重新分析原始数据并补了六组秒级探针,分析链如下。

### 5.1 数据的真实形状是阶跃,不是曲线

把 318 个真实激活测量画出来:漂移只有三簇——精确 0、~1e-5 量级、~3e-3 平台,中间几乎没有过渡点。

```
漂移(log 尺度)
3e-3 ┤              ●●●●●●●●●●●   ← 饱和平台(常数)
3e-5 ┤        ●●                  ← 稀疏的过渡点
   0 ┤ ●●●●●                      ← 精确零
     └────────────────────────→ 任何连续几何量
```

对"两级台阶+常数平台"做连续回归:平台内 y 是常数而 x 在变,R² 必然低;零值点在 log 空间无法表示。**最佳 R²=0.48 不是"没规律",是"规律不是回归能表达的形状"。** S1a 的失败是判据错配,责任在计划设计(预注册了错误的函数形式),不在数据。

S1b 的 L35 失败同理:D=256→4096 平台内 6% 的反序是 ulp 级噪声,过渡区的单调性(0→0→3e-5→2.3e-3→平台)真实而陡峭,判据对"平台并列"缺乏容忍。

### 5.2 平台 = BF16 输出舍入天花板

平台配置实测 35–47% 输出元素恰差 1 个最低位 → 平台高度 ~3e-3 ≈ BF16 格距(2⁻⁸≈0.004)。推论:**单层内"置换多少"不是变量**(F3 散置 5% 与 50% 的中位漂移同为 ~2.9e-3)。

### 5.3 关键实验:判据是对齐,不是距离

原数据显示"位移 ≤8 全部逐比特零",像是"位移小就安全"。补充探针推翻了这个读法:

| 探针 | 位移 | torch BF16 | vLLM batch-invariant |
|---|---|---|---|
| 偶对齐成对交换 (0,1)(2,3)… | 1 | **0(逐比特)** | **0(逐比特)** |
| **奇对齐**成对交换 (1,2)(3,4)… | **1** | **2.7e-3(天花板!)** | 4.3e-5 |
| 对齐 8-窗内乱序 | ≤7 | **0(逐比特)** | **0(逐比特)** |
| 8-窗偏移 4(跨块) | ≤7 | 2.8e-3 | 4.2e-5 |
| 对齐 16-窗内乱序 | ≤15 | ~4e-5 | ~5e-5 |

torch 探针中,奇对齐的 distance-1 交换已能达到与全局打乱相同的单层天花板;这说明**位移小或跨块坐标少都不能保证漂移小**,但并非声称任意一次跨块都必然到顶。**免费的充分条件:∀i,i 与 π(i) 同属一个对齐 8-块(⌊i/8⌋ 相同)。** 原计划所有小位移构造碰巧全是对齐的,才产生"位移定律"的假象。

### 5.4 免费类跨 kernel、跨形状、端到端成立

免费类置换在 torch GEMM 与 vLLM batch-invariant kernel、序列长度 T∈{1,124,512,2048,8192}、三个层上全部逐比特为零。天花板高度则是 kernel 属性:vLLM batch-invariant 的 prefill 天花板比 torch 低约 17 倍(1.9e-4 vs 3.2e-3)——"kernel 依赖"从理论提醒升级为已测量事实。

### 5.5 过渡区不可精确预测(两个自然假说均被否定)

F2 连续块族出现同几何、不同 seed 漂移差 1436 倍的反例。复审测试了两个假说:被移动坐标的激活能量份额(R² 仅 0.03–0.30)、被移动的最大激活幅度(判别准确率 91.5%)——都不足以精确预测过渡区。残差由 kernel 归约微结构 × 数据对齐决定。**对设计无妨:待在免费档即可,不需要预测过渡区。**

### 5.6 连带勘误:上一实验的一个结论被推翻

`ffn_benchmark_eval` 曾结论"相邻交换的 benchmark 影响 ≈ 随机的 1/4"。但相邻交换是偶对齐的,按 §5.3 应当零影响。回查原始数据:其 6 个 benchmark 中 **5 个不一致率恰好为 0**,唯一非零的 GSM8K(7.2%)完全落在 instruct 模型自身的噪声地板内(11 个同权重 baseline 两两对比:均值 6.0%、最大 7.8%)。**"1/4"是把噪声当成了效应;相邻交换与零效应完全一致。** 已在该实验 RESULT.md 追加勘误 §8。这也反过来印证了免费类在完整推理引擎 + benchmark 层面成立。

## 6. 修正后的定律与对原始问题的回答

**单层 BF16 down_proj 漂移三档表**(Qwen3-4B-Base,RTX 4090):

| 档位 | 条件 | torch bf16 | vLLM batch-invariant |
|---|---|---|---|
| 免费 | 保持对齐 8-块成员 | 0(逐比特) | 0(逐比特) |
| 中间 | 仅对齐 16~32 窗内混合 | ~3e-5 – 1e-4 | ~3e-5 – 1e-4 |
| 饱和 | 实质跨块 | 2.4e-3 – 3.2e-3(BF16 ulp 天花板) | ~2e-4(prefill)/~6e-5(decode) |

**"要置换 30% neurons,怎么组织代价最小?"**

1. 若置换可以设计为**块内重排/配对**(每个被动的 neuron 只在自己的对齐 8-块内移动)→ 代价**严格为零**,逐比特、端到端。
2. 若必须跨块 → 不能用"跨块越少,代价越小"来连续优化;稀窄过渡区依赖 kernel 微结构和数据对齐,一旦形成实质性跨块则通常饱和,跨多少、跨多远不再提供可靠排序 → 更可靠的剩余杠杆是**动哪些层**(上游已证:只动第 0 层 ≈ 全 36 层效应的 75%,只动最后一层 ≈ 0)。
3. 所以"好置换"不是连续优化问题,而是:**离散的免费类判定 + 层选择**。

**边界与 caveat**:块宽 8 和天花板高度是当前两个 backend(torch bf16 GEMM、vLLM 0.24 batch-invariant)在 RTX 4090 上的实测属性,不是自然常数;换 kernel/硬件需用预留的 `backend.py` 接口重测,不得直接外推。

### 6.1 重要认知修正:实质跨块近似开关,不是斜坡

这是读完三档表后**最容易产生、且已实际发生过的误读**,单独记录如下。

**误读**:"跨块(块宽 8)的 neuron 越少,漂移越小;所以设计目标是尽量少跨块。"——听起来自然,但**在单层内不成立**。

**数据**:F3 全局散置族,同一层、同一 backend(torch bf16):

| 跨块坐标数 | rel_l2 中位数 |
|---|---|
| ~486(K=5%) | ~2.9e-3 |
| ~973(K=10%) | ~2.9e-3 |
| ~2918(K=30%) | ~2.9e-3–3.2e-3 |
| ~4864(K=50%) | ~2.9e-3–3.2e-3 |
| 9728(全局随机) | ~2.9e-3–3.2e-3 |

跨块数量差 20 倍,漂移不变。机制(§2.3):跨块扰动的效果是让每个输出元素最后一位的舍入方向近似掷硬币,天花板由 BF16 输出格距决定,与扰动了多少项无关。

**正确的表述**:

- 单层内,跨块是**二值开关**:零跨块 = 严格免费(逐比特);实质跨块 = 直接到天花板。中间只有一条极窄的过渡带(极少量低幅度坐标跨块时),它依赖数据且已被验证**无法用先验指标精确预测**(§5.5),不能当设计旋钮用。
- 因此"少跨块"作为**连续优化目标没有意义**;有意义的目标只有"**零跨块**"(进免费类)这一个离散判定。
- 真正连续的剂量轴在**层间**:多少个层处于非免费状态、这些层在多深的位置(L0≈全模型效应的 75%,L35≈0)。要控制模型级代价,调的是这个轴,不是单层内的跨块比例。

一句话:**单层内没有可靠的"半个代价"可买;工程上可控的主要是免费与全价两档,更稳定的剂量调节在层的维度上。**

## 8. 第二轮执行(修订案 v1.1)结果与下一步

原阶段二至四作废后,`AMENDMENT_v1.1.md` 预注册了三个新阶段(1b 单层确认 / 2b 引擎端到端逐比特 / 3b benchmark 三臂终验)。codex 已执行:

| backend × 形状 | 免费类逐比特 | 非免费漂移结构 |
|---|---|---|
| torch bf16 GEMM(T≥2) | ✅ 126/126(加上探针两轮合计 504/504) | 紧天花板 ~2.9e-3(p95/p5 ≤ 2.2) |
| vLLM batch-invariant prefill | ✅ 126/126 | 弥散小带 2e-5–2e-4 |
| vLLM batch-invariant decode(T=1) | ✅ 126/126 | 弥散小带 |
| **torch bf16 GEMV(T=1)** | ❌ 53/126(**73 条失败全部在此**) | **无块结构、无天花板**:任何置换只有 ≤6e-5、偶发 1–3/2560 元素的微噪声 |

要点:

- 73 条失败每条只有 1–3/2560 个输出元素不同,rel_l2 ∈ [1.1e-9, 6.0e-5]——是"第三种归约微结构"(GEMV 跨步分组,无连续 8-块),量级上无害,但逐比特保证失效;
- 失败原因是修订案作者(reviewer)把 torch-GEMV 纳入了免费预测适用域——此前的单点探针恰好没触发翻位,5-seed × 2-prompt 网格把它测了出来;这是规格错误,免费类定律本身在部署栈(vLLM 两种形状)上 **252/252 无一例外**;
- torch GEMM 的紧天花板得到预注册确认;vLLM 的"饱和档"实为宽约 10 倍的弥散小带,"三档"在 vLLM 上应读作"零 / 非零小带"两档。

**下一步(v1.2,待定)**:把免费类硬判据的适用域改为三个已验证组合,torch-GEMV 单列为"微噪声区"(软判据 ≤1e-4);vLLM 饱和档改为区间判据;恢复执行 2b/3b(其硬判据只依赖 vLLM 栈,预测不变)。另有两项已讨论未预注册的扩展:剂量-响应链接(复用 ffn_benchmark_eval 的 8 个锚点 checkpoint + 补测 32-prompt logits,零新 benchmark 成本)和"波动还是退化"的 seed 过拟合检验(纯分析,零 GPU 成本,详见与用户 2026-07-12 的讨论记录)。
